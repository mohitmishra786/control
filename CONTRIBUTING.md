# Contributing to Control

Thank you for your interest in contributing to Control. We welcome contributions from the community to help make this tool better for everyone.

## Getting Started

1. Fork the repository.
2. Clone your fork: `git clone https://github.com/your-username/control.git`
3. Create a branch for your changes: `git checkout -b feature/your-feature-name`
4. Make your changes and ensure they follow the project's coding standards.
5. Commit your changes: `git commit -m "Description of your changes"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Create a Pull Request against the main repository.

## Coding Standards

- Use Swift naming conventions (PascalCase for types, camelCase for functions and variables).
- Document all public APIs using triple-slash (`///`) comments.
- Ensure all tests pass before submitting a Pull Request.
- Follow the architectural patterns established in the project.

## Development Setup

To build the project locally, you will need:

- macOS 12.0+
- Swift 6.0+
- Xcode (optional, but recommended for debugging)

Run `swift build` from the root directory to build the project.

## Reporting Issues

If you find a bug or have a feature request, please search the existing issues before creating a new one. When reporting an issue, include:

- A clear description of the problem.
- Steps to reproduce the issue.
- Your macOS version and device architecture (Intel or Apple Silicon).
- Any relevant logs or error messages.
